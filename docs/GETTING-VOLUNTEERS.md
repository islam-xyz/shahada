# How to Get Volunteers Helping on Shahada.org

This doc is for **you as the maintainer**. It explains how to reach the level where volunteers actively help make the website better — like Quran.com and similar projects.

---

## How sites like Quran.com get volunteers

1. **They make the repo the single source of work** — All tasks (bugs, features, translations) live in GitHub Issues. Contributors find work by browsing issues and pull requests.
2. **They point people to one place** — A clear "Developers" or "Contribute" page on the live site that links to GitHub, a short "how to help," and sometimes a "get in touch" form.
3. **They label easy entry points** — "Good first issue" and "Help wanted" so new contributors can find something to do without asking.
4. **They respond** — When someone opens an issue or PR, maintainers reply. That builds trust and encourages more contributions.
5. **They keep the bar low** — Clear CONTRIBUTING.md, simple tech stack (e.g. static HTML), and one-change-per-PR so reviews are fast.

---

## What you already have (in place)

- **README** with mission, contributing section, and link to CONTRIBUTING
- **CONTRIBUTING.md** with clear ways to help and rules
- **Code of Conduct** and **Security** policy
- **Issue templates**: Bug, Feature request, Translation
- **PR template** for consistent descriptions
- **Open license** and "open to contributions" messaging

## What to do next (checklist)

### 1. Use GitHub labels consistently

- Create (if missing) and use these labels in your repo:
  - **good first issue** — Small, well-defined task (e.g. fix one typo, update one translation string).
  - **help wanted** — You’d like community help (e.g. "Review this language", "Improve this section").
- When you open a new issue, add the right label so people can filter:
  - [Open good first issues](https://github.com/islam-xyz/shahada/issues?q=is%3Aissue+is%3Aopen+label%3A%22good+first+issue%22)
  - [Open help wanted](https://github.com/islam-xyz/shahada/issues?q=is%3Aissue+is%3Aopen+label%3A%22help+wanted%22)

### 2. Keep 3–5 "good first issue" tasks open

- Add 2–3 real issues and label them **good first issue** (e.g. "Fix typo in X language", "Improve one FAQ answer", "Add meta description for Y page").
- When one gets done, add another. A steady list of small tasks makes it easy for new people to start.

### 3. Link to contribution from the live site

- Add a **Contribute** (or **Developers**) page on the site (e.g. `shahada.org/en/contribute` or `shahada.org/contribute`) that:
  - States the mission in one sentence.
  - Lists ways to help (translations, bugs, accessibility, content).
  - Links to: GitHub repo, Issues, "Good first issue" filter, CONTRIBUTING.md.
- Add a "Contribute" link in the footer (or header) of the main page so visitors can find it.

### 4. Respond quickly to issues and PRs

- Aim to reply to every new issue within a few days (even if it’s "Thanks, we’ll look at this").
- Review and merge or give feedback on PRs within a week when possible. Slow response is the main reason contributors leave.

### 5. Optional: "I want to help regularly"

- Like Quran.com’s form: add a simple Google Form (or GitHub Discussion) where people can say "I want to volunteer my time" and optionally mention skills (translations, dev, design).
- You can then reach out when you have a matching task. Link this form from the Contribute page and README.

### 6. Recognize contributors

- In README, add a short "Contributors" section or link to GitHub’s Contributors tab.
- Optionally thank recent contributors in release notes or a short "Thank you" on the Contribute page. Recognition encourages repeat contributions.

---

## Summary

| Action | Why it helps |
|--------|------------------|
| Label issues "good first issue" / "help wanted" | People can find work without asking. |
| Keep a few small issues open | New contributors have a clear first step. |
| Contribute page on the site | Visitors and potential volunteers see one clear entry point. |
| Fast, friendly responses | Builds trust and encourages more PRs. |
| Optional volunteer form | Lets you match "I want to help" with tasks later. |

You don’t need a big team or budget. You need: **visible tasks**, **clear links from the website to GitHub**, and **consistent, kind responses**. That’s how you get to the level where volunteers help make Shahada.org better.
